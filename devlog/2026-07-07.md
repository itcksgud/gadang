## 1. Docker화 (P1)

**구성**
- `Backend/Dockerfile`: 멀티스테이지 (maven:3.9-temurin-17 빌드 → temurin-17-jre 실행).
  pom.xml 먼저 복사해 의존성 레이어 캐시. `.dockerignore`로 `application-local.properties`
  제외 — 시크릿은 이미지에 굽지 않고 compose 환경변수로 주입.
- `ai-server/Dockerfile`: python:3.12-slim + uvicorn. corpus.json/embeddings.npy 포함(재임베딩 없이 기동).
- `docker-compose.yml`: mysql(healthcheck+volume) / redis:7-alpine / ai-server / backend.
  - 컨테이너 간 통신은 서비스명 DNS (`mysql`, `redis`, `ai-server`).
  - `SPRING_DATASOURCE_URL` 등 env가 properties보다 우선(relaxed binding) → 코드 수정 없이 환경 전환.
  - depends_on: mysql `service_healthy` 이후 backend 기동.

**호스트 포트 충돌 조정** (다른 프로젝트 컨테이너·로컬 MySQL과 공존)
- mysql 3307→3306, redis 6380→6379, ai-server 8001→8000. backend는 8080 그대로.

**검증**: 4컨테이너 기동, `Started BackendApplication`, `/api/places/regions` 200,
ai-server `/health` = `{"status":"ok","docs":32,"indexed":true}`.

**삽질 기록**
- Docker Desktop 엔진 미기동 상태에서 build → pipe 에러. CLI 존재 ≠ 엔진 실행.
- 반쯤 뜬 이전 컨테이너가 포트 점유 → `docker compose down` 후 재기동.
- Git Bash에서 curl 한글 파라미터가 EUC-KR로 전송돼 400/500 → UTF-8 퍼센트 인코딩으로 호출해야 함.

## 2. Redis L2 캐시 (P4)

**설계**: L1 Caffeine(인스턴스 로컬) 유지, L2를 DB 테이블(REGION_PLACES) → Redis로 교체.
`RealScoredPlaceProvider`가 이미 JSON 문자열로 직렬화하고 있어 `StringRedisTemplate`로
자연 치환 — 직렬화 설정 불필요.

**변경**
- pom: `spring-boot-starter-data-redis` 추가
- properties: `spring.data.redis.host=${REDIS_HOST:localhost}` / `port=${REDIS_PORT:6380}`
  (로컬 실행은 호스트 6380 = 컨테이너 redis, 컨테이너 실행은 compose env로 redis:6379)
- `RealScoredPlaceProvider`: `ExternalCacheMapper`(DB) → `StringRedisTemplate`
  - 저장: `opsForValue().set(key, json, Duration.ofDays(7))` — **TTL을 Redis EXPIRE에 위임**
    (기존엔 SQL `updated_at > now-7d` 수동 검사)
  - Redis 장애 시 try/catch로 우회(캐시 없이 동작) — graceful degradation
- 테스트 `RealScoredPlaceProviderTest`: DB mock → StringRedisTemplate/ValueOperations mock. 2/2 통과.
- 참고: TREND_CACHE·REGION_IMAGE·TRANSIT_ROUTE 등 다른 L2는 아직 DB — 추후 이관 후보.

**실측 (region=전주, radius=3000)**
| 상태 | 응답시간 | 경로 |
|---|---|---|
| 콜드 (캐시 없음) | **22.4s** | 카카오 22구역 + 네이버 블로그 수백 건 실호출 |
| 2차 호출 | **0.07s** | L1 Caffeine 히트 |
| 백엔드 재시작 직후 | **0.68s** | L1 소멸 → **Redis L2 히트** (`[Place/L2] ... redis hit (200 places)`) |

Redis 키 확인: `places:v10:716,2543:ALL:전주`, TTL 604,774s(≈7일).
→ "재시작·다중 인스턴스에서도 살아있는 공유 캐시" 실증. 자소서/면접용 수치 확보.
