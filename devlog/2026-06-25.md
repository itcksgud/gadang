## 2. STS 서버 기동 오류 수정

팀원이 pull한 `TagoTrainService`가 `tagoTrainRestClient` 빈을 필요로 하는데 제거된 상태였음.

**원인**: 이전에 TAGO → 코레일 API 전환 시 `tagoTrainRestClient` 빈 삭제했지만, 팀원 코드가 여전히 참조.

**해결**: `RestClientConfig.java`에 호환 빈 재추가 (실제로는 코레일 URL 사용).

```java
@Bean("tagoTrainRestClient")
public RestClient tagoTrainRestClient() {
    return RestClient.builder().requestFactory(timeoutFactory())
            .baseUrl("https://apis.data.go.kr/1613000/TrainInfoService")
            .build();
}
```

## 3. 로그인 / 회원가입 버그 수정

### 3-1. 401 이후 UI 상태 불일치

**증상**: 토큰 만료 후 API는 실패하지만 헤더는 로그인된 상태 유지.

**원인**: `api/index.js`의 401 인터셉터가 `localStorage`만 지우고 모듈 레벨 reactive ref(`token`, `user`)는 건드리지 않음.

**해결**:
- `api/index.js` → 401 시 `window.dispatchEvent(new Event('auth:logout'))` 발행
- `useAuth.js` → 모듈 레벨에서 이벤트 수신, ref 초기화

```js
// useAuth.js 모듈 레벨
window.addEventListener('auth:logout', () => {
  token.value = ''
  user.value  = null
})
```

### 3-2. 회원가입 에러 메시지 항상 이메일 오류로 표시

**원인**: catch 블록이 모든 에러를 "이미 사용 중인 이메일" 하드코딩.

**해결**: `e?.response?.data?.message` 추출 후 fallback 처리.

### 3-3. 회원가입 후 자동 로그인

회원가입 성공 시 바로 login() 호출 → `/home` 리다이렉트.

## 4. 소셜 로그인 (네이버 + 카카오)

### 아키텍처: 프론트엔드-callback 방식

```
브라우저 → OAuth 제공자 (로그인 UI)
         ← code 반환 (프론트 callback URL로)
프론트   → POST /api/auth/{provider}/exchange {code, redirectUri}
백엔드   → 제공자에서 토큰 교환 + 프로필 조회 → JWT 발급
```

네이버 개발자 콘솔에 `http://localhost:5173/auth/naver/callback` 이 등록되어 있어서 프론트 callback 방식으로 결정. client_secret은 백엔드에만 있음.

### 백엔드 추가 내용

- `OAuthController.java`: `POST /api/auth/naver/exchange`, `POST /api/auth/kakao/exchange`
- `OAuthService.java`: RestClient.create()로 토큰 교환 + 프로필 조회
- `schema.sql`: USER 테이블에 `provider`, `social_id` 컬럼 추가, `email`/`password` nullable 처리
- `User.java`, `UserMapper.java`: provider/socialId 필드 + `findBySocialId` 메서드
- `SecurityConfig.java`: exchange 엔드포인트 permitAll 추가

카카오는 이메일이 선택 권한이라 없을 수 있음 → `kakao_{id}@social.gadang.local` 합성 이메일 사용.

### 프론트엔드 추가 내용

- `LoginView.vue`: 네이버(초록), 카카오(노랑) 버튼. OAuth URL로 직접 리다이렉트.
- `OAuthCallbackView.vue` (신규): `/auth/naver/callback`, `/auth/kakao/callback` 처리.
  - code 받아서 백엔드 exchange API 호출
  - JWT 저장 후 원래 페이지로 복귀
- `router/index.js`: callback 라우트 추가
- `.env.local`: `VITE_NAVER_CLIENT_ID`, `VITE_KAKAO_CLIENT_ID`

state 파라미터에 로그인 후 돌아갈 경로 인코딩해서 OAuth 왕복 후에도 복귀.

## 5. 코스 생성 → 로그인 → 이어서 진행

**문제**: 코스 만들고 일정 확정 누르면 로그인 페이지로 넘어가는데, 로그인 후 코스 데이터가 사라짐.

**해결**: `sessionStorage`에 코스 데이터 저장 → 로그인 후 `CourseView` 마운트 시 복원.

```js
// 로그인 전 저장
sessionStorage.setItem('pendingCourse', JSON.stringify(result.value))
router.push('/login?redirect=/course')

// 로그인 후 복원 (onMounted)
const saved = sessionStorage.getItem('pendingCourse')
if (saved && isLoggedIn.value) {
  result.value = JSON.parse(saved)
  sessionStorage.removeItem('pendingCourse')
}
```

## 6. 게시글 작성 확장 — 머리말 / 꼬리말 / 금액 / 소요 시간

### 목표

사용자가 여행 후기를 쓸 때 장소별 **실제 지출 금액**과 **소요 시간**을 남기면, 그 데이터가 DB에 쌓여 나중에 장소 통계로 활용 가능.

### DB 변경

```sql
-- COMMUNITY_POST 신규 컬럼
ALTER TABLE COMMUNITY_POST ADD COLUMN IF NOT EXISTS intro             TEXT;
ALTER TABLE COMMUNITY_POST ADD COLUMN IF NOT EXISTS outro             TEXT;
ALTER TABLE COMMUNITY_POST ADD COLUMN IF NOT EXISTS total_cost        INT NOT NULL DEFAULT 0;
ALTER TABLE COMMUNITY_POST ADD COLUMN IF NOT EXISTS total_duration_min INT NOT NULL DEFAULT 0;

-- 장소별 상세 데이터 적재 테이블
CREATE TABLE IF NOT EXISTS POST_PLACE_DETAIL (
    detail_id    BIGINT AUTO_INCREMENT PRIMARY KEY,
    post_id      BIGINT NOT NULL,
    seq          INT    NOT NULL DEFAULT 0,
    place_id     BIGINT NULL,          -- PLACE 테이블 FK (카카오 장소)
    place_name   VARCHAR(100) NOT NULL,
    text_content TEXT,
    cost         INT NOT NULL DEFAULT 0,   -- 해당 장소 지출 (원)
    duration_min INT NOT NULL DEFAULT 0,   -- 소요 시간 (분)
    images       TEXT,                     -- JSON 배열 ["url1","url2"]
    FOREIGN KEY (post_id) REFERENCES COMMUNITY_POST(post_id) ON DELETE CASCADE
);
```

`place_id`로 같은 장소의 여러 후기를 집계 가능:

```sql
-- 장소별 평균 비용/소요 시간 집계 예시
SELECT place_name,
       AVG(cost)         AS avg_cost,
       AVG(duration_min) AS avg_duration_min,
       COUNT(*)          AS review_count
FROM POST_PLACE_DETAIL
WHERE place_id = ?
GROUP BY place_name;
```

### 백엔드 변경

| 파일 | 내용 |
|---|---|
| `PostRequest.java` | `intro`, `outro`, `List<PlaceDetail> places` 추가. PlaceDetail에 `cost`, `durationMin`, `images` 포함 |
| `PostPlaceDetail.java` | 신규 엔티티 |
| `PostPlaceDetailMapper.java` | insert / findByPostId / deleteByPostId |
| `PostResponse.java` | `intro`, `outro`, `totalCost`, `totalDurationMin`, `List<PlaceDetailResponse>` 추가 |
| `CommunityMapper.java` | INSERT/UPDATE/SELECT에 신규 컬럼 반영 |
| `CommunityService.java` | 총 금액·시간 자동 계산, POST_PLACE_DETAIL 저장 |

### 프론트엔드 변경

**PostWriteView.vue**
- 머리말 textarea (최상단)
- 꼬리말 textarea (최하단)
- 장소별: 💰 금액 입력 (원) + ⏱ 소요 시간 (시간/분)
- TOC 사이드바에 총 지출·총 소요 시간 실시간 합계 표시
