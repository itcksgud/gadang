# 교통 시간표 DB 저장 구조 협업 가이드

> 작성일: 2026-06-24  
> 목적: 홈 탭에서 교통 정보 조회 시 시간표를 DB에 함께 저장하고, 코스 탭에서 활용하는 방법 정리

---

## 1. 전체 데이터 흐름

```
홈 탭 (지역 검색)
  └─ RegionDiscoveryService.discoverByRail()
       └─ TagoTrainService.fetchAndStore()
            ├─ TRANSIT_ROUTE 에 집계값 저장 (기존)
            └─ TRANSPORT_SCHEDULE 에 편별 시각 저장 (신규)

홈 탭 (지역 검색)
  └─ RegionDiscoveryService.discoverByBus()
       └─ OdsayBusService.getScheduleByIds()
            └─ OdsayBusService.saveSchedule()
                 └─ TRANSPORT_SCHEDULE 에 편별 시각 저장 (신규)
```

---

## 2. 기존 테이블: `TRANSIT_ROUTE`

**용도**: 집계값 캐시 (운행 횟수, 최저요금, 최단시간)  
**키**: `(from_hub, to_hub, type)`  
**TTL**: 기차 20시간, 버스 6일

| 컬럼 | 설명 |
|------|------|
| from_hub | 출발역/터미널명 (버스는 `ODSAY:터미널ID`) |
| to_hub | 도착역/터미널명 |
| type | `KTX` / `무궁화` / `ITX-청춘` / `시외버스` |
| duration_min | 최단 소요시간(분), -1=미운행 |
| fare | 최저 편도 요금(원) |
| daily_trips | 하루 운행 횟수 |

---

## 3. 신규 테이블: `TRANSPORT_SCHEDULE`

**용도**: 편별 출발·도착 시각 (코스 탭에서 몇 시 열차/버스 탈지 선택할 때 사용)  
**키**: `(from_hub, to_hub, type, dep_time, travel_date)`

| 컬럼 | 설명 |
|------|------|
| from_hub | 출발역/터미널명 |
| to_hub | 도착역/터미널명 |
| type | `KTX` / `무궁화` / `ITX-청춘` / `시외버스` |
| dep_time | 출발 시각 `HH:mm` |
| arr_time | 도착 시각 `HH:mm` (null 가능) |
| duration_min | 소요시간(분) |
| fare | 편도 요금(원) |
| train_no | 열차번호 (버스는 NULL) |
| travel_date | 기준 날짜: 기차=당일, 버스=`1970-01-01` (정적) |

### travel_date 규칙
- **기차**: 당일 날짜 (`LocalDate.now()`). 홈에서 지역 조회할 때마다 당일 시간표로 갱신.
- **버스**: 항상 `1970-01-01` 고정. 버스 시간표는 거의 변하지 않아 날짜 무관 정적 취급.

## 4. 데이터 소스별 파싱 정보

### 기차 (TAGO API)
- **API**: `GET /GetStrtpntAlocFndTrainInfo`
- **원본 필드**: `depplandtime` (`yyyyMMddHHmmss`), `arrplandtime`, `adultcharge`, `traingradename`, `trainno`
- **저장 변환**: `depplandtime` → `HH:mm` (`parseHHmm()`)
- **TTL**: 20시간 (당일 시간표이므로 짧음)

### 버스 (Odsay API)
- **API**: `GET /searchInterBusSchedule`
- **원본 필드**: `startTime` (`HHMM`), `endTime`, `wasteTime`(소요분), `fare`
- **저장 변환**: `startTime` `HHMM` → `HH:mm` (`busHHmm()`)
- **TTL**: 6일 (정적 시간표)
- **주의**: `from_hub`가 터미널 이름(예: "서울고속버스터미널")으로 저장됨

---

## 5. 조회 API

### 특정 수단 시간표 조회
```
GET /api/transport/schedule?fromHub={역명}&toHub={역명}&type={KTX|무궁화|시외버스}
GET /api/transport/schedule?fromHub=서울역&toHub=신경주역&type=KTX
```
응답: `TransportScheduleEntry[]` (dep_time 오름차순)

### 전체 수단 시간표 조회 (기차+버스 합산)
```
GET /api/transport/schedule/all?fromHub={역명}&toHub={역명}
GET /api/transport/schedule/all?fromHub=서울역&toHub=신경주역
```
응답: `TransportScheduleEntry[]` (type → dep_time 순)

## 6. 코스 탭에서 활용 방법

### 현재 흐름
1. 홈 탭에서 지역+교통수단 선택 → `selectedTransport` 쿼리 파라미터로 코스 탭 이동
2. 코스 탭은 `transportOneWayMin`, `transportFare` 등 집계값만 받음
3. 백엔드에서 `SelectedTransport`로 코스 타임라인 생성 시 "몇 시 열차"는 미정

### 앞으로 코스 탭에서 구체적 시각 활용하려면
```js
// 코스 탭 프론트: 시간표 API 호출
const schedules = await fetch(
  `/api/transport/schedule?fromHub=${selectedTransport.fromHub}&toHub=${selectedTransport.toHub}&type=${selectedTransport.mode}`
).then(r => r.json())
// 사용자가 원하는 편 선택 → dep_time을 코스 요청에 포함
```

백엔드 `CourseRequest`에 `transportDepTime` 필드 추가하면, `addSelectedOutbound()`에서 해당 시각으로 출발 시각 고정 가능.

## 8. 주의사항

- **TRANSIT_ROUTE 기본키 충돌 없음**: `TRANSPORT_SCHEDULE`은 별도 테이블, 기존 집계 흐름 영향 없음
- **버스 from_hub 형식**: `TRANSIT_ROUTE`는 `ODSAY:터미널ID`, `TRANSPORT_SCHEDULE`은 터미널 이름(fetchSchedule에서 `fromName`으로 전달)
- **Odsay API `startTime` 필드**: API가 항상 반환을 보장하지 않음. `null`이면 해당 편 저장 스킵
- **기차 당일 시간표**: 홈에서 지역 검색할 때마다 기존 당일 데이터 삭제 후 재삽입 (편 취소/변경 반영)
